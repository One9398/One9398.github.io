发表于 2016-07-05   |   分类于 编程   |  

Window 与 View

在iOS中, Window对象只是作为View对象的容器,本身没有任何可视化的内容展示在屏幕上.而屏幕上展示的文字,图片,形状等都是在View对象中添加的,然后将View对象添加到Window对象上,让视图内容展现到屏幕上,并且View对象间可以进行嵌套和组合形成一定的视图层次关系让最终呈现内容更加复杂.

iOS程序一启动,Appdelegate对象都初始化一个主Window对象,一般我们配置视图后都被添加在主Window对象,然后呈现到屏幕上.

View

一个View就是UIView或者其子类的实例对象,主要负责了三个方面: 内容绘制, 事件处理, 布局管理

  • 内容绘制: 在iOS中绘制View所采用的绘制技术主要Core Graphic, OpenGL ES, UIKit.
  • 事件处理: 响应各种手势事件,Touch事件
  • 布局管理: 使用frame/bound 或者autolayout来进行自己subViews中各视图的位置和尺寸设置, 父视图可以动态地调整子视图布局来响应各种条件变化如动画,旋转等

View and Window Architecture

Window

一个Window就是UIWindow的实例,处理iOS程序的用户界面在设备上整体的呈现,管理和交互添加在Window的View对象,来改变屏幕的呈现内容.常规地,程序中大多时候都只存在一个Window对象用于内容的展示,也可以创建一个的Window对象并且加入当前的Window层次中,添加View内容后呈现在屏幕上.

Animations

动画主要行为就是用可视化的反馈效果来表现View层次变化或者自身样式的变化,主要的动画技术有UIKit动画, Core Animation.
