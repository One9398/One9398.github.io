View Controller Programming Guide For iOS (一)

发表于 2016-07-25   |   分类于 编程

控制器总览

1.控制器的作用

iOS中ViewControllers分两种:

  • 内容控制器,管理自己根视图上的内容.
  • 容器控制器,管理子控制器和控制子控制器间的切换


1.1视图管理

对于内容控制器来说,控制器管理着属于自己根视图的所有层次视图,控制器通过对其根视图和所有子视图进行强引用来管来每一个视图对象.
而容器控制器只管理自己的根视图以及子控制器的根视图尺寸和位置,不管理子控制器上的视图内容,如系统的容器控制器SplitViewController

1.2数据处理

控制器还管理视图与模型之间的数据处理,将模型数据进行一定加工装载到视图上.但是控制器处理模型数据时要尽量减少对模型数据的管理,应让模型对象自身拥有数据处理的逻辑,然后由控制器进行执行,避免控制器代码的臃肿.


1.3用户交互

作为响应者对象,也具有基本的处理响应链中传递的事件的能力,但多数情况下视图都应自己处理对于的触摸事件或者使用delegate或target-action将事件转交给控制器处理.

1.4资源管理

控制器持有着所有在其根视图上的视图(包括根视图),以及创建的其他对象,所有这些对象的内存都由控制器分配管理.在手机内存紧张时,系统会自动地调用控制器的didReceiveMemoryWarning方法进行临时资源内存的释放,通常可以在此方法中手动释放一些懒加载的对象以及其他不重要但内存占量大的对象.

1.5适应多尺寸

控制器负责视图的应对各个尺寸环境下的呈现方式,比如iPhone的横屏与竖屏两种不同尺寸:水平方向和垂直方向上的Regular Size和Compact Size,控制器应使用Autolayout来合理地布局内容到对应尺寸上,进行布局的动态调整.

2.控制器层次

2.1根控制器

根控制器是控制器层次中的最末端,也是Window对象rootViewController属性指向的对象,来提供初始的视图内容添加到当前的可视窗口上.


2.2容器控制器

容器控制器结合多个子控制器的自定义视图内容组成最终界面来呈现内容,为此需要合理地放置子控制器根视图以及恰当地填充整个容器控制器的根视图界面.

2.3呈现视图控制器

为了在当前控制器下呈现出另一个新的控制器来展示新的内容,常常使用默认的Modal Presentied方式来表现,其中presentedViewContoller表示着呈现出来的新控制器对象,presentingViewController为原来界面的控制器对象.

当有容器控制器存在情况下进行呈现新控制器操作时,UIKit提供合适的上下文来表现:默认选择跟当前控制器层次关系最近的容器控制器来presented新控制器,通常也是窗口的根控制器,当然可允许调整控制器所定义的呈现上下文,进行手动地呈现控制.
