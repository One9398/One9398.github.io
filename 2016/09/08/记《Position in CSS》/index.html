记《Position in CSS》

发表于 2016-09-08   |   分类于 编程   |  

Basic

  • CSS属性position不会被继承
  • 绝对定位的元素会脱离正常文档流, 并生成Block-level box.

Offset

  • 常用属性:left,right,top,bottom 描述定位元素在Containing Block各个边的偏移量.
  • 每个属性都单独针对一个边

Width and Height

  • 浮动元素都设置了有效的left,right,top,bottom后能自动计算出元素的宽高.

Contain Overflow and Clipping

针对元素的内容超过元素盒子尺寸,超出的内容需要隐藏时,设置overflow为hidden.

Element Visibility

使用visibility属性可以元素隐藏,但是元素仍在正常的文档流布局中占据位置,与设置display为none的效果不同.

Absolute Positioning

  • 绝对定位元素的offset设置会基于自己的Containing Block.(离自己最近祖先元素position不为static的元素; 如果不存在则指定为Viewpoint)

  • 当元素被绝对定位后,会创建一个Containing Block来约束它的后代元素的布局行为

  • 定位元素使用auto-margin实现垂直居中.

  • 定位元素使用transform: translate(-50%, y);实现水平居中

Note:如何居中布局定位元素

  • 浮点Non-Replaced元素的宽度计算公式 (高度同理)

Fixed Postioning

行为和作用与 Absolute Positioning一样,只是Containing Block指定为 Viewpoint.

Relative Postioning

  • 相对定位元素的布局不会脱离正常文档流, 并且offset基于自身原始的位置变化,偏移后仍占有原来的位置,显示空白.
  • 通常应用场景为:将元素进行相对定位,创建一个Containing Block提供给绝对定位的子元素进行布局.(子绝父相)

Addition

Replace Element
能根据其标签的元素与属性来判断显示具体的内容,具有内部的宽高.
a replaced element is an element whose representation is outside the scope of CSS, eg:img,video,form element(input, textarea).More Detail

Block-level box
用来包容后代box和内容;
Block-level boxes are boxes that participate in a block formatting context. Each block-level element generates a principal block-level box that contains descendant boxes and generated content and is also the box involved in any positioning scheme. Some block-level elements may generate additional boxes in addition to the principal box: ‘list-item’ elements. These additional boxes are placed with respect to the principal box.

More Detail

Viewpoint
浏览器可视窗口点,以左上角为原点(0,0).
