我的2017年目标清单

发表于 2017-02-02   |   分类于 成长   |  

总结之后,更重要的是为新一年列出一份目标清单, 保证自己下一年的前进方向.很幸运,最近从吴军老师的一篇文章中学到了如何利用谷歌的目标管理法(OKR)来给自己设定目标.

OKR: Google的每个员工，每个季度都会给自己定一个或者几个目标Objectives，并且衡量目标是不是能达成关键结果Key Results，这几个词合在一起被称为OKR。每个人的OKR会放到自己的网页上，大约半页纸长，大家都可以看到。当季度结束时，每个人会给自己的目标完成情况打分，完成了得分是1，部分完成的话，得分是0到1之间的一个数字，没完成的得分就是0。Google强调每个人制定的目标要有挑战性，所以如果谁完成目标的情况总是1，并不能说明他工作好，而是目标定的太低。大部分情况下，大家完成的目标都在0.7－0.8左右, 并且每当总结工作时, 可以增加当初制定的目标, 对于不打算完成的目标，或者已经过时不再有意义的目标，不能删除，但是可以说明为什么没有做。

作为一种个人的目标管理方式, 我也尝试来为新一年的自己所前行方向设定了目标和关键结果, 经过一小时多的思考,我暂时想到了编程, 阅读, 锻炼, 英语这四个最具重要意义的主要目标,也对应想出了与这些目标相关的关键结果,把这些记录下来去行动, 而不是说说,想想而已.

目标1: 完成 Java 课程,掌握好 Java 企业开发技能

关键结果

  • 将学习过程中的每日总结和笔记整理成小册子
  • 在北京找一份合适的开发工作

目标2: 认真读完10本书,其中选择3本外文书

关键结果

  • 为认真读过的书整理一张内容相关的思维导图
  • 将所认真读完书后的收获和感想整理写一篇文章,发表到自己公众号上
  • 对外文书进行大致翻译,整理成小册子

目标3: 坚持锻炼运动,慢跑,俯卧撑

关键结果

  • 慢跑达成200公里
  • 俯卧撑做满3000个

目标4: 模仿朗读提高自己英语口语

关键结果

  • 参与清晨朗读会的文章口语按照提供的方法练习50篇以上
  • 将口语练习中的生词,陌生语法,常用词组记录,整理成小册子
  • 进行一次出国旅行,用英语与他人交流

上面是我目前能够想到的四个目标, 所有目标的关键结果都需要自己真正地行动, 才会真正让自己产生变化, 不然又只是一次大脑在制定计划时的自我满足, 可能有些地方实践起来会遇到困难, 以及到后面可能会对目标和结果进行调整, 但我仍会朝着完成所设定的目标方向去努力, 至少也要去完成了其中的70%, 在2018年来到之前,回来看看这些文字. 是始终如一地践行, 还是厚颜无耻地打脸, 自己到底会做出怎样的选择呢, 期待那一天吧.

离2018年还有333天, 恩就这样开始吧 :]

《Object-Oriented JavaScript》问答

发表于 2016-10-08   |   分类于 编程   |  

本文是针对Object-Oriented JavaScript一书的阅读记录,前后看了两遍想到尝试用问答的形式帮助自己记忆和巩固一些Javascript中面向对象的知识点,也方便以后对此内容的回顾,书上的部分内容现在还不能完全理解,只能靠自己实践中去积累过后再去挖掘了.如果发现哪里有描述不对的地方,欢迎交流.

Primitive Data Types

Q: 请说出Javascript的五个基本数据类型

A: Number, String, Boolean, Undefined, Null

Q: typeof与instanceof分别的作用是什么?

A: typeof用来查看某个变量的数据类型,结果都是字符串形式如”number”,”string”, “undefined”, “object”, “function”等;而instanceof只是用来验证某个对象是否为特定对象构造方法创建的实例对象.

Q: Javascript中字符串表示的数字使用运算符时一定会被转换成数字进行计算吗?

A: 不一定,因为+在字符串中也有特定的意义, 当运算符是+时,只会被看做字符串进行拼接,而不进行运算操作.

Q: Javascript的表示false的值有哪几种?

A: 六种;false, “”, undefined, 0, NaN, null.

Q: Undefined 与 Null 区别

A: Undefined表示不存在的变量或者未定义的变量;而Null用来表示空的值,只有定义了才会出现.

iOS10新变化之废弃的openURL

发表于 2016-09-25   |   分类于 编程   |  

前言

作为UIApplication单例对象的方法openURL:在iOS开发中经常用来实现在当前应用打开外部链接的需求比如跳转到其他应用,跳转应用隐私设置界面;还有相关APIcanOpenURL:进行跳转前判断.而这个古老(iOS2时出现)的openURL 方法将被现今iOS10出现的新APIopenURL:options:completionHandler:所替代.

虽然关于该API的官方文档补充完全,但所提供的信息足够能让开发者简单了解使用最新API打开外部链接的API究有何异同.

新API的官方描述

首先来看官方文档对iOS10的What’s New in iOSUIKit部分对新API的描述

The new UIApplication method openURL:options:completionHandler:, which is executed asynchronously and calls the specified completion handler on the main queue (this method replaces openURL:).

这段话清楚地指明了新API的两个特点:异步执行open操作和主线程执行回调.

Chapter 1:Scope

Basic

  • 编写JavaScript程序时,JavaScript作为动态解释型语言, 同样是先编译成可执行的代码(JavaScript引擎优化使得编译速度极快),然后再执行.
  • JavaScript程序的编译过程由编译器先一步完成,其次执行过程由JavaScript引擎完成.


运行Javascript程序的三要素

  • JavaScript引擎 : 负责整个过程的编译,以及程序的执行.
  • 编译器 : 处理词法解析和代码生成的过程
  • 作用域 : 收集和持有一个向上查找的列表,其包含声明过的变量;以及设定了各个变量的访问范围的规则.
